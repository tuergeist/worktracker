## Der Pivot
User: „kein Raw-SQL, es muss ein Auth-Framework geben" + „immer erst Library suchen + fragen, dann bauen". → Stack via context7 recherchiert, Plan vorgelegt, Freigabe geholt. Umgesetzt:
- **fastapi-users** (Google OAuth via httpx-oauth, Cookie+JWT, integer ids), **async SQLAlchemy 2.0** ORM.
- Alle Domain-Endpoints async + `current_active_user`-geschützt; `user_id` kommt aus der Session (raus aus Bodies/Queries).
- Frontend: Auth-Gate (`auth.js`), Vollbild-Google-Login bis eingeloggt; Settings zeigt Account + Abmelden; Spieler-Wechsel entfernt.

## Entscheidungen (Kontext)
- **Google-only Accounts**, Login Pflicht, Spieler-Profile entfallen. Später weitere Social-Auths / user+pass / E-Mail-OTP (fastapi-users kann OAuth+Passwort; OTP = späteres Custom-Add-on).
- Timestamps bewusst weiter als UTC-**TEXT** (`to_char`) → Frontend-Kontrakt (`new Date(v+"Z")`) bleibt.
- `RedirectCookieTransport`: überschreibt `get_login_response` → 302 auf `/` (sonst leere 204-Seite nach OAuth-Callback).

## Stolperfallen (wichtig!)
- **OAuthAccount-FK**: `SQLAlchemyBaseOAuthAccountTable` setzt FK auf `user.id`, unsere Tabelle heißt `users` → Relationship findet keinen Join. Fix: `user_id` per `@declared_attr` auf `users.id` überschreiben.
- **is_default Boolean vs altes INTEGER**: altes `pgdata`-Volume (psycopg-Phase) → „argument of IS TRUE must be type boolean". Fix: `docker compose down -v` (DB-Reset war freigegeben). Bei Schemawechsel immer Volume droppen — `create_all` altert keine Tabellen.
- **`#app{display:flex}` schlägt `[hidden]`** (ID-Spezifität) → Header leakte über Login. Fix: `#app[hidden]{display:none}`.
- fastapi-users leitet die OAuth-Callback-URL per Request ab → hinter TLS-Ingress `--proxy-headers` nötig, damit `https` greift. Redirect-Pfad ist `/api/auth/google/callback` (nicht `/api/auth/callback`).
- Static-Files sind ins Image gebacken → Frontend-Änderungen brauchen `docker compose up -d --build` (restart reicht nicht).
