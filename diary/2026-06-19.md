Fortsetzung des Mobile-first-Umbaus → Deployment-Reife.

## Was passiert ist
- **Frontend-Umbau** auf die Mocks fertig (Shell/Putten/Range via Subagents, sequenziell
  gegen einen DOM-Contract, um Datei-Konflikte zu vermeiden). Backend-API blieb unverändert.
- **Range-Eingabe**: Distanz-Buckets als Primär-Input + Exakt-Slider (Step 5 m). Buckets und
  Slider grauen sich gegenseitig aus (nur visuell, weiter bedienbar zum Umschalten).
- **Putten Green-Foto**: neuer Endpoint `POST /api/analyze-putt`, nutzt `tools/putt-analyzer`
  (Claude-VLM + CV). Liefert Zonen/Tendenz/Streuung + annotiertes PNG. UI: Kamera-Button →
  Ergebnis-Sheet. putt-analyzer-Tool ins Repo committet (venv/Caches via .gitignore raus).
- **Docker/Deploy**: Compose, dann GitHub Actions (GHCR multi-arch) + k8s-Manifeste.
- **DB-Migration SQLite → Postgres** (Entscheidung des Users: „SQLite causes only trouble").
  psycopg3 + dünner Connection-Wrapper, der `?`-Platzhalter und `lastrowid` (via `RETURNING id`)
  übersetzt → **app.py-Call-Sites unverändert**. Zeitstempel bewusst als UTC-**TEXT** gespeichert
  (`to_char(now() AT TIME ZONE 'UTC', ...)`), damit der JSON-Kontrakt (`new Date(v+"Z")`) gleich bleibt.
  Connect-Retry beim Start für k8s/Compose-Reihenfolge.

## Entscheidungen / Kontext (nicht aus git ableitbar)
- **Range-Datenmodell** weicht bewusst von EXPERIENCE.md ab: Buckets statt großem Zahlen-Input.
  `carry_m` = Slider/Override sonst Bucket-Mittelwert; `drift_m` = Richtung (−1/0/+1, keine Magnitude
  ohne Trackman). Range-Stats zeigen daher keine Meter-Streuung, sondern Tendenz.
- **App läuft auf Port 8080**, nicht 8000 — Host-8000 ist von `sportboot-app` belegt.
- k8s: App ist zustandslos (2 Replicas), Postgres als StatefulSet (RWO, 1 Replica). Kein SQLite-PVC mehr.

## Stolperfallen (für später)
- **`lsof -ti:8000 | xargs kill -9` hat die OrbStack-VM mitgerissen** — der getroffene Prozess war
  OrbStacks host-seitiger Port-Forward-Listener auf 8000. Nie wieder kill auf 8000; Port nur lesen.
- **Docker-Disk lief voll** („No space left on device" beim pip-Build, numpy/scipy schwer).
  `docker builder prune -af` + `image prune -af` → 3,5 GB frei. Volumes fremder Projekte nicht angefasst.
- `uvicorn` ist nicht im PATH, nur als `python3 -m uvicorn`.
- zsh: `UID` ist read-only — nicht als Shell-Var nutzen.
